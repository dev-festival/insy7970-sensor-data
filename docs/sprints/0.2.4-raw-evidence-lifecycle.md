# Sprint 0.2.4 - Raw Evidence Lifecycle

## Problem Statement

Live Waites pulls are valuable evidence, but the raw payloads grow quickly. A single day can be tens of megabytes as plain JSON, and repeated canaries or backfills will become noisy unless raw evidence has a disciplined lifecycle. This sprint defines and implements the "scraps" side of the hewing machine: preserve proof long enough to validate and reprocess, compress it cheaply, verify it later, and prune it only through explicit commands.

## Requirements

1. Record raw artifact byte counts, compressed byte counts, SHA-256 checksums, and compression state in `manifest.json`.
2. Add `sensor-data raw compress --source waites --date YYYY-MM-DD` to gzip raw Waites JSON files after validation without changing their logical artifact identity.
3. Add `sensor-data raw verify --source waites --date YYYY-MM-DD` to confirm manifests, checksums, required files, and compressed artifacts are readable.
4. Add `sensor-data raw prune --source waites --older-than-days N --dry-run` with dry-run default behavior and explicit confirmation for deletes.
5. Keep raw manifests small, secret-safe, and readable without loading large payload files.
6. Support both uncompressed `.json` and compressed `.json.gz` raw artifacts in downstream readers.
7. Document default retention guidance for live raw evidence versus long-lived processed outputs.

## In Scope

- Gzip compression for raw Waites artifacts.
- Manifest checksums, byte counts, compression metadata, and verification status.
- CLI commands for compress, verify, and prune.
- Storage helpers for locating `.json` or `.json.gz` forms of the same logical artifact.
- Tests with small fixture files and `tmp_path`.
- Documentation for evidence retention and safe pruning.

## Control Boundary

This sprint adds operator controls, not a policy engine. The controls are explicit CLI commands that let a maintainer compress, verify, and prune raw evidence deliberately. It does not add scheduled retention, automatic deletion, background workers, or production storage administration.

The default posture should be conservative:

- `compress` is safe and repeatable.
- `verify` is read-only.
- `prune` defaults to dry-run and requires an explicit non-dry-run flag before deleting anything.
- No command deletes raw evidence unless a manifest exists and verification can explain what is being removed.

## Out of Scope

- SQLite observation loading.
- Replacing snapshot or trend processing with database queries.
- Automatic background cleanup.
- Object storage, cloud buckets, or network shares.
- Compression formats beyond gzip.
- Deleting raw files without an explicit command.

## Test Out Plan

- Run `uv run pytest`.
- Test compression writes `.json.gz`, removes or marks superseded `.json` according to the chosen contract, and updates manifest metadata.
- Test verification passes for both plain JSON and gzip JSON artifacts.
- Test verification fails clearly when checksum, byte count, or required artifact files do not match the manifest.
- Test prune dry-run lists candidate dates without deleting anything.
- Test prune execution only deletes date directories older than the requested threshold.
- Test downstream raw readers can load compressed and uncompressed artifacts through the same helper.

## Implementation Notes

Endpoint `path` values remain the logical `.json` artifact identity. Sprint `0.2.4` adds an `artifact` block to each endpoint manifest entry to describe the current storage file, compression state, byte counts, and SHA-256 checksums. Plain endpoint JSON can be replaced by `.json.gz`, but downstream code should continue asking for the logical `.json` path.

The raw lifecycle command family is:

```powershell
uv run sensor-data raw verify --source waites --date YYYY-MM-DD
uv run sensor-data raw compress --source waites --date YYYY-MM-DD
uv run sensor-data raw prune --source waites --older-than-days N
uv run sensor-data raw prune --source waites --older-than-days N --delete --confirm-delete
```

`verify` is read-only. `compress` validates first and is repeatable. `prune` defaults to dry-run and only deletes verified candidates when `--delete --confirm-delete` is provided.
