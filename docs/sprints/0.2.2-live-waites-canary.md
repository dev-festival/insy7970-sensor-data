# Sprint 0.2.2 - Live Waites Canary

## Problem Statement

The mock pipeline is useful, but it should not drift far from the real Waites API shape. This sprint adds a narrow, opt-in live Waites fetch path that saves raw evidence exactly like mock mode while keeping normal tests and development independent of credentials, network access, and large datasets.

## Requirements

1. Implement `sensor-data waites fetch --source api --date YYYY-MM-DD --facility N` using `WAITES_BASE_URL` and `WAITES_ACCESS_TOKEN` from `.env`.
2. Use the same request-building contract and raw output paths as mock mode.
3. Save raw live responses under `data/raw/waites/date=YYYY-MM-DD/` without downstream transformation.
4. Write a manifest that includes endpoint names, request parameters, response status or error details, record counts, elapsed time, and output paths without storing secrets.
5. Fail clearly when the Waites token is missing, expired, unauthorized, or when the API response shape is unsupported.
6. Add opt-in live tests or smoke checks that are skipped unless explicitly enabled.
7. Document safe live-data usage, including small date ranges and the fact that `data/` remains ignored by Git.

## In Scope

- Thin Waites HTTP client behavior.
- API-mode raw evidence capture.
- Manifest hardening for live runs.
- Secret-safe logging and summaries.
- Opt-in live test marker or CLI smoke command.
- Docs for running the canary with local `.env` actuals.

## Out of Scope

- Long-range live backfills.
- Live Maximo/DB2 access.
- Live trend or clustering quality claims.
- Persisting credentials outside `.env`.
- Automatic retries beyond a conservative first pass.
- Web UI changes.

## Test Out Plan

- Run `uv run pytest` with all live tests skipped by default.
- Test API-mode fetch refuses to run without a configured Waites token.
- Test HTTP request construction with a fake transport or mocked client.
- Test live response envelopes are written to the same raw artifact paths used by mock mode.
- Test manifests redact or omit authorization secrets.
- Add an opt-in command or pytest marker such as `live` for manual validation against real Waites access.

## Next Sprint Lead In

Sprint `0.2.3` uses canary output to validate the raw response contract and prove that live raw files can flow into the existing snapshot and trend builders without rewriting downstream logic.
