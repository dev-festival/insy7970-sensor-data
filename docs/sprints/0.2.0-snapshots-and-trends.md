# Sprint 0.2.0 - Snapshots, Trends, and Read API

## Problem Statement

Raw sensor readings are not yet usable for equipment review. This sprint turns mock Waites evidence into processed daily sensor snapshots and simple trends by equipment, sensor, and installation point, then serves those outputs through FastAPI. This creates the first working browser-accessible version of the project with mock data.

## Requirements

1. Build daily sensor snapshots grouped by `installation_point_id` using mean, standard deviation, minimum, and maximum statistics for available readings.
2. Summarize RMS readings by axis for acceleration, velocity, peak-to-peak, and crest factor.
3. Summarize ImpactVue acceleration and temperature readings into the same snapshot row shape.
4. Join installation point and equipment metadata so processed outputs preserve names, `equipment_id`, `sensor_id`, and `customer_asset_id`.
5. Add `sensor-data snapshot build --date YYYY-MM-DD --source mock` to write CSV and metadata files under `data/processed/snapshots/`.
6. Add `sensor-data trend build --start-date YYYY-MM-DD --end-date YYYY-MM-DD --source mock` to write lightweight trend-ready outputs under `data/processed/trends/`.
7. Add FastAPI endpoints for available dates, daily snapshots, and trends, plus enough static app wiring to view mock snapshot and trend data.

## In Scope

- DataFrame transformations from raw mock readings.
- Processed CSV and JSON metadata outputs.
- Unit conversion rules carried forward from the reference project, with units documented in output metadata.
- CLI commands for snapshot and trend generation.
- FastAPI read endpoints for snapshot and trend artifacts.
- Minimal static app flow backed by FastAPI responses.
- Basic validation for missing or partial endpoint files.
- README quickstart showing the full mock workflow.

## Out of Scope

- KMeans clustering.
- Cluster drift.
- Real Waites API requests.
- Maximo/DB2 data.
- Full dashboard polish.
- User profiles or saved preferences.

## Test Out Plan

- Run `uv run pytest` with small fixture files and expected output columns.
- Test aggregation for ImpactVue, RMS by axis, and temperature independently.
- Test unit conversion behavior with known input values.
- Test metadata joins preserve asset alignment fields.
- Test CLI snapshot and trend commands using `tmp_path` as the data directory.
- Test FastAPI date, snapshot, and trend endpoints with `TestClient`.
- Test graceful failure when a required raw file is missing.
